# 100+ CCNA Interview Questions and Answers

## 1. What is the OSI model?
### Answer:
The OSI model is a conceptual framework used to understand network communication. It consists of 7 layers:
1. **Physical Layer**: Deals with the physical connection between devices.
2. **Data Link Layer**: Provides node-to-node data transfer.
3. **Network Layer**: Handles routing of data packets across networks.
4. **Transport Layer**: Ensures reliable data transfer.
5. **Session Layer**: Manages sessions between applications.
6. **Presentation Layer**: Translates data into a format the application can understand.
7. **Application Layer**: Provides networking services to end-user applications.

### Real-World Scenario:
In a network troubleshooting session, a technician might refer to the OSI model to determine the layer at which a communication error occurs.

### Follow-Up Questions:
- Can you explain the differences between TCP and UDP?
- Which OSI layers are involved in TCP communication?

## 2. What is a VLAN?
### Answer:
A VLAN (Virtual Local Area Network) is a logical subdivision of a network that groups devices into a single broadcast domain, regardless of their physical location.

### Real-World Scenario:
A company may use VLANs to separate network traffic for different departments, ensuring better security and performance.

### Follow-Up Questions:
- How does VLAN tagging work?
- Can you explain the difference between a private and a public VLAN?

## 3. Explain the function of a router.
### Answer:
A router is a device that forwards data packets between computer networks, creating an overlay network. It operates at Layer 3 of the OSI model, using IP addresses to determine the best path for forwarding the packets.

### Real-World Scenario:
In a home network, the router enables multiple devices to connect to the Internet by sharing a single public IP address.

### Follow-Up Questions:
- What is NAT and how does it work?
- Can you describe routing protocols and their importance?

---

## 4. What is the difference between a switch and a hub?
### Answer:
A switch is a networking device that receives incoming data packets and redirects them to their destination on the local area network. A hub, on the other hand, is a basic device that connects multiple Ethernet devices, making them act as a single network segment.

### Real-World Scenario:
In an office, using switches improves network performance compared to hubs as they reduce collisions and improve bandwidth efficiency.

### Follow-Up Questions:
- Can you explain how a switch learns MAC addresses?
- How does a switch handle broadcast traffic?

---

## 5. What is DHCP?
### Answer:
DHCP (Dynamic Host Configuration Protocol) is a network management protocol used to automate the process of configuring devices on IP networks, allowing them to use DHCP to obtain IP addresses and other related configuration parameters automatically.

### Real-World Scenario:
When a new device connects to a corporate network, the DHCP server assigns it a temporary IP address, enabling immediate access to network resources.

### Follow-Up Questions:
- What is the difference between DHCP and static IP addressing?
- Can you explain how a DHCP relay works?
